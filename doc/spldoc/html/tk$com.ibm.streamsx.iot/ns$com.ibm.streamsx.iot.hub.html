IoT message hub connectivity.

Message hub application

The recommended approach for this toolkit is to implement connectivity to the IoT message hub in a single application that publishes topics for device events, device commands and optionally device statuses. This separates message hub specific connectivity from analytics against devices.

Analytic applications then subscribe to these topics using EventsSubscribe, CommandsSubscribe, and StatusesSubscribe.

Analytics applications can publish device commands to be sent to devices using CommandPublish. The message hub application subscribes to the topic and is then responsible for ensuring the device command is delivered.

IoT topics

Table 1. Message hub topics

Topic

Type

Description

streamsx/iot/device/events

DeviceEvent

device events sent by devices - published by hub application, subscribed to by analytic applications using EventsSubscribe.

streamsx/iot/device/commands/sent

DeviceCmd

device commands sent to devices - published by hub application, subscribed to by analytic applications using CommandsSubscribe.

streamsx/iot/device/commands/send

DeviceCmd

Send a device command to a device - published by analytic applications using CommandPublish and subscribed to by message hub application. The message hub application is responsible for sending the device onto the message hub for delivery to the device.

streamsx/iot/device/statuses

DeviceStatus

device statuses - published by hub application, subscribed to by analytic applications using StatusesSubscribe. Optional if the message hub does not provide device status events.
